Welcome to my Project Euler solution page. Project Euler is hundreds of increasingly difficult mathematical problems intended to be solved with computer programs. I enjoy solving Project Euler in Scala. This page is a web front-end to my solutions to many of Project Euler Problems. This page's backend is in Scala using Play Framework and Akka actors. It is deployed in a Docker container running on Amazon EC2 Container Service (ECS) which is part of Amazon AWS. This page itself is HTML with CSS and Javascript and jQuery. It uses Ajax to retrieve problem solutions and Websockets to receive a continuous stream of system status updates from the backend. Each of the numbered buttons you see on the right hand of the screen is linked to a corresponding Project Euler problem. There is a link to the problem below and to the right of the button. Clicking a button assigns one of the backend's Akka actors to solving that problem. The button turns red while solution is in progress and a timer below and to the left of the button counts seconds till the answer is received. When the solution is received the button turns blue and the answer is displayed below it. The two gauges on the left represent the continuously updated state of the backend. The top gauge shows the percent of all available memory that's being utilized in the Java Virtual Machine (JVM) within which the Scala backend runs. This information is streamed from the backend to the front-end via a Websocket connection. The bottom gauge shows the responsiveness of the backend, even as it's solving many Project Euler problems simultaneously. The front end expects to receive status updates from the backend every second. That corresponds to 100% responsiveness. If that expectation is not met, the responsiveness gauge will decline from the full 100%. Many of Project Euler problems are extremely memory- and CPU-intensive, especially the higher-numbered ones. The backend runs on a single AWS "micro" instance with a single virtual CPU and just one gigabyte of RAM. It is not hard to overwhelm the backend by simultaneously starting the solutions for many higher-numbered problems by clicking on corresponding buttons. In those cases the backend will eventually run out of memory and "crash" and its container will then be automatically restarted by ECS.
